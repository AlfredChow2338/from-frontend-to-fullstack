## Questions

1. How do you return 404 for unknown id — which exception type?
2. Why return a response DTO instead of the raw entity?

---

## Answers

1. In Nestjs, to return 404 for not finding targeted resources can `throw new NotFoundException('Not found resources')`

2. Entities often carry a lot more information than actually needed. This is performance consideration. DTO shapes API more efficiently.

### Reviewer comments

- **Q1:** Correct exception choice: use `NotFoundException` for unknown resource ids, usually from the service layer after lookup returns null/undefined. Nice that you used concrete Nest syntax. Small polish: message should be singular and specific (e.g., `Note with id 123 not found`) for easier client debugging.
- **Q2:** Good direction: response DTOs enforce API contract and avoid leaking internal fields. Beyond performance, the bigger wins are **security** (hide sensitive/internal columns), **decoupling** (DB schema can evolve without breaking API), and **consistency** for clients.
