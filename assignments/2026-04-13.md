**Theme:** Node fundamentals

## Questions

1. What is the Node.js LTS policy in one sentence, and which `node -v` output proves your install?
2. Why does `npm init -y` create a `package.json`, and what two fields identify the project?

## Answers

1. LTS policy means Long-term supprt, which the version will be maintained to fix crtical bugs for 30 months.

`node -v` output the version of nodejs runtime proves the installation eg. `v22.18.0`

2. `npm init` will ask for number of config questions to create a new `package.json` project. Flag `-y` simply means all default config options.

The package name and the version identifies the project.

### Reviewer comments

- **Q1:** Your LTS summary (~30 months of maintenance) matches how Node describes LTS support windows. Using `node -v` to confirm the binary is installed is correct. Small polish: spell “support” / “critical”; if you want one extra layer, look up **Current** vs **LTS** lines on [Node.js releases](https://nodejs.org/en/about/previous-releases).
- **Q2:** Correct: `npm init -y` skips the questionnaire and writes defaults into `package.json`. The two identifying fields are **`name`** and **`version`** (often written as `name` / `version` in the file).
