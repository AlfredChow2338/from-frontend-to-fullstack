## Questions

1. Sketch your standard JSON error shape and when you’d include `details`.
2. How do you avoid leaking internal errors to clients in production?

---

## Answers

1. Using RFC 7807 (Problem Details for HTTP APIs) ensures consistent error handling

```json
{
  "status": 400,
  "code": "VALIDATION_ERROR",
  "instance": "/users/1/update",
  "details": {
    "field": "email",
    "issue": "invalid format",
    "value": "abc@gmail"
  }
}
```

2.

- intercept error and convert to standard JSON format
- use general error message eg. Internal Server Error to replace the exact sensitive errors eg. SQLException
- avoid revealing details such as schema name and table names
- expose `errorId` please contact administrator with the id to get more information internally
