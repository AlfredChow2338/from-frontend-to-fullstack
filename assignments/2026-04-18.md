**Theme:** Node fundamentals

## Questions

1. List two things that are _not_ the same between browser JS and Node (APIs or globals).
2. In your own words: what does “single-threaded event loop” mean for CPU-heavy work?

---

## Answers

1. Browser has a top-level window object which provides access to Document Object Model.

Browser storage such as localStorage sessionStorage and cookies method are available.

Node environment has global object but no window object because of no visual interface.

Node's global process object provides information about the execution environment such as process.env and process.exit().

2. Everything is waiting on the queue.

Operation freeze: In browser, buttons are not clicked. In Node, requests are not process until the heavy calculation finishes.

The event loop stuck and cannot move tasks forward.

Using worker thread or web worker can process the heavt calculation work to run on separate thread to avoid current blocking.

### Reviewer comments

- **Q1:** Good contrast between browser and Node globals/APIs (`window`/DOM/storage vs `global`/`process`). Nice job giving concrete examples instead of only definitions.
- **Q2:** Core idea is correct: CPU-heavy work blocks the single main thread, so the event loop cannot advance other callbacks/requests. Small wording fix: not everything is waiting in one queue; tasks are scheduled across phases, but progress still stalls while sync CPU work runs.
