## Questions

1. What is the purpose of the `scripts` field in `package.json`? Give an example `npm run` you added.
2. What is the difference between `dependencies` and `devDependencies`?

---

## Answers

1. `scripts` is an object that you can add custom commands to automate process in your project. For example:

```json
{
  "scripts": {
    "dev": "next dev",
    "build": "next build"
  }
}
```

`npm run dev` and `npm run build`

2. `dependencies` needed in production environment, eg. lodash, react. `devDependencies` need in developing and building the projects eg. jest, eslint, prettier, webpack.

### Reviewer comments

- **Q1:** Correct purpose: named npm scripts for repeatable tasks. Small fix applied in-file: the field in `package.json` is **`scripts`** (plural), not `script`. Optional: mention `npm run` passes through extra args with `--`.
- **Q2:** Correct split: `dependencies` ship with production/runtime installs; `devDependencies` are for local dev, tests, and build tooling (often not installed in production when using `npm install --omit=dev`).
